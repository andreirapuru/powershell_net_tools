Introdução

Este guia é voltado para iniciantes que desejam aprender a usar o PowerShell para gerenciar e diagnosticar redes. Ele cobre 30 ferramentas e comandos, desde os tradicionais, como ping e ipconfig, até cmdlets modernos do PowerShell, como Get-NetAdapter e Test-NetConnection. Cada seção inclui uma explicação detalhada do conceito por trás do comando, exemplos práticos de uso, saídas esperadas e, quando aplicável, informações específicas sobre Wi-Fi ou placas de rede.

O PowerShell é uma ferramenta poderosa para administradores de sistemas, oferecendo maior flexibilidade e capacidade de automação em comparação com o Prompt de Comando (CMD). Este guia foca em comandos úteis para tarefas de rede, como verificar conectividade, configurar interfaces, monitorar tráfego e gerenciar configurações de Wi-Fi. As saídas fornecidas são exemplos simulados baseados em resultados típicos, ajudando a entender o que esperar ao executar cada comando.

Ferramentas de Rede

1. Ping

Conceito: O comando ping utiliza o protocolo ICMP (Internet Control Message Protocol) para testar a conectividade entre o computador local e um destino remoto (identificado por IP ou hostname). Ele envia pacotes Echo Request e aguarda respostas Echo Reply, medindo a latência (tempo de resposta) e verificando perdas de pacotes. É amplamente usado para diagnosticar problemas de conectividade, como verificar se um servidor está online ou se há falhas na rede. O comando é simples, mas não oferece muitas opções de personalização no CMD.

Exemplo:

ping google.com

Exemplo de Saída:


Pinging google.com [142.250.190.78] with 32 bytes of data:
Reply from 142.250.190.78: bytes=32 time=25ms TTL=117
Reply from 142.250.190.78: bytes=32 time=24ms TTL=117
Reply from 142.250.190.78: bytes=32 time=26ms TTL=117
Reply from 142.250.190.78: bytes=32 time=25ms TTL=117

Ping statistics for 142.250.190.78:
    Packets: Sent = 4, Received = 4, Lost = 0 (0% loss),
Approximate round trip times in milli-seconds:
    Minimum = 24ms, Maximum = 26ms, Average = 25ms

Exemplo com parâmetros:

ping google.com -n 10

Explicação: O parâmetro -n 10 especifica que 10 pacotes serão enviados, em vez dos 4 padrão, permitindo um teste mais prolongado.

2. Test-Connection

Conceito: O cmdlet Test-Connection é a versão PowerShell do ping, oferecendo maior flexibilidade e integração com pipelines. Ele também usa ICMP para testar conectividade, mas retorna objetos PowerShell, permitindo filtragem, formatação ou automação avançada. Pode especificar a interface de origem, testar múltiplos destinos simultaneamente e ajustar parâmetros como tamanho do pacote ou tempo de espera. É ideal para scripts de monitoramento de rede.

Exemplo:

Test-Connection -ComputerName google.com -Count 4

Exemplo de Saída:


Source        Destination     IPV4Address      IPV6Address  Bytes    Time(ms)
------        -----------     -----------      -----------  -----    --------
DESKTOP-ABC   google.com      142.250.190.78                32       25
DESKTOP-ABC   google.com      142.250.190.78                32       24
DESKTOP-ABC   google.com      142.250.190.78                32       26
DESKTOP-ABC   google.com      142.250.190.78                32       25

Exemplo com Wi-Fi:

Test-Connection -ComputerName 192.168.1.1 -Source (Get-NetAdapter -Name "Wi-Fi").Name

Exemplo de Saída:


Source        Destination     IPV4Address      IPV6Address  Bytes    Time(ms)
------        -----------     -----------      -----------  -----    --------
Wi-Fi         192.168.1.1     192.168.1.1                   32       5
Wi-Fi         192.168.1.1     192.168.1.1                   32       4
Wi-Fi         192.168.1.1     192.168.1.1                   32       6
Wi-Fi         192.168.1.1     192.168.1.1                   32       5

Explicação: Testa a conectividade com o gateway local usando a interface Wi-Fi, útil para verificar a estabilidade da conexão sem fio.

3. Netstat

Conceito: O comando netstat exibe informações detalhadas sobre conexões de rede ativas, portas abertas, tabelas de roteamento e estatísticas de interface. Ele é útil para monitorar quais aplicativos estão usando a rede, identificar portas em uso e diagnosticar problemas de conectividade. No PowerShell, cmdlets como Get-NetTCPConnection oferecem funcionalidades semelhantes, com saídas estruturadas para automação.

Exemplo:

netstat -ano

Exemplo de Saída:


Active Connections

  Proto  Local Address          Foreign Address        State           PID
  TCP    0.0.0.0:135            0.0.0.0:0              LISTENING       896
  TCP    192.168.1.100:49152    142.250.190.78:443     ESTABLISHED     4568
  UDP    0.0.0.0:123            *:*                                    1024

PowerShell alternativo:

Get-NetTCPConnection | Select-Object LocalAddress, LocalPort, RemoteAddress, RemotePort, State

Exemplo de Saída:


LocalAddress    LocalPort RemoteAddress    RemotePort State
------------    --------- -------------    ---------- -----
192.168.1.100   49152     142.250.190.78   443        Established
127.0.0.1       5357      0.0.0.0          0          Listen

Explicação: Lista conexões TCP ativas, útil para identificar tráfego de rede ou portas abertas.

4. Get-NetAdapter

Conceito: O cmdlet Get-NetAdapter lista todas as placas de rede físicas e virtuais no sistema, incluindo adaptadores Wi-Fi, Ethernet e virtuais (como os usados em VPNs). Ele fornece informações como nome da interface, status (conectado/desconectado), endereço MAC, velocidade do link e tipo de mídia. É essencial para diagnosticar problemas de hardware de rede ou configurar interfaces específicas.

Exemplo:

Get-NetAdapter

Exemplo de Saída:


Name            InterfaceDescription                    ifIndex Status
----            --------------------                    ------- ------
Wi-Fi           Intel(R) Wireless-AC 9560               12      Up
Ethernet        Realtek PCIe GbE Family Controller      8       Disconnected

Exemplo Wi-Fi:

Get-NetAdapter -Name "Wi-Fi"

Exemplo de Saída:


Name            InterfaceDescription                    ifIndex Status
----            --------------------                    ------- ------
Wi-Fi           Intel(R) Wireless-AC 9560               12      Up

Explicação: Exibe detalhes apenas da placa Wi-Fi, útil para verificar se está ativa.

5. Ipconfig (Get-NetIPConfiguration)

Conceito: O comando ipconfig exibe configurações de rede, como endereço IP, máscara de sub-rede, gateway padrão e servidores DNS. No PowerShell, Get-NetIPConfiguration é seu equivalente, oferecendo uma saída estruturada e mais fácil de manipular em scripts. É usado para verificar configurações de rede de todas as interfaces ou de uma específica, como Wi-Fi.

Exemplo:

ipconfig

Exemplo de Saída:


Windows IP Configuration

   Wireless LAN adapter Wi-Fi:
      Connection-specific DNS Suffix  . : home
      IPv4 Address. . . . . . . . . . . : 192.168.1.100
      Subnet Mask . . . . . . . . . . . : 255.255.255.0
      Default Gateway . . . . . . . . . : 192.168.1.1

PowerShell alternativo:

Get-NetIPConfiguration

Exemplo de Saída:


InterfaceAlias       : Wi-Fi
InterfaceIndex       : 12
IPv4Address          : 192.168.1.100
IPv4DefaultGateway   : 192.168.1.1
DNSServer            : 8.8.8.8

Exemplo Wi-Fi:

Get-NetIPConfiguration -InterfaceAlias "Wi-Fi"

Exemplo de Saída:


InterfaceAlias       : Wi-Fi
InterfaceIndex       : 12
IPv4Address          : 192.168.1.100
IPv4DefaultGateway   : 192.168.1.1
DNSServer            : 8.8.8.8

Explicação: Mostra configurações de IP apenas para a interface Wi-Fi, útil para verificar a conectividade sem fio.

6. Arp (Get-NetNeighbor)

Conceito: O comando arp gerencia a tabela ARP (Address Resolution Protocol), que mapeia endereços IP para endereços MAC na rede local. É usado para resolver endereços na camada de enlace (Layer 2) e pode ajudar a identificar dispositivos na mesma sub-rede. No PowerShell, Get-NetNeighbor fornece uma saída mais estruturada, ideal para automação.

Exemplo:

arp -a

Exemplo de Saída:


Interface: 192.168.1.100 --- 0xc
  Internet Address      Physical Address      Type
  192.168.1.1           00-14-22-01-23-45    dynamic
  192.168.1.2           00-16-17-AB-CD-EF    dynamic

PowerShell alternativo:

Get-NetNeighbor

Exemplo de Saída:


IPAddress         PhysicalAddress      InterfaceAlias
---------         ---------------      --------------
192.168.1.1       00-14-22-01-23-45    Wi-Fi
192.168.1.2       00-16-17-AB-CD-EF    Wi-Fi

Explicação: Lista dispositivos na rede local, úteis para troubleshooting de conectividade na sub-rede.

7. Tracert (Test-NetConnection)

Conceito: O comando tracert (traceroute) rastreia o caminho que os pacotes percorrem até um destino, mostrando cada roteador (salto) no trajeto e a latência associada. É útil para diagnosticar onde ocorrem atrasos ou falhas na rede. No PowerShell, Test-NetConnection -TraceRoute oferece funcionalidades semelhantes, com saída estruturada.

Exemplo:

tracert google.com

Exemplo de Saída:


Tracing route to google.com [142.250.190.78]
over a maximum of 30 hops:

  1     5 ms     4 ms     5 ms  192.168.1.1
  2    10 ms    11 ms    10 ms  10.0.0.1
  3    20 ms    21 ms    20 ms  172.16.0.1
  4    25 ms    24 ms    25 ms  142.250.190.78

Trace complete.

PowerShell alternativo:

Test-NetConnection -ComputerName google.com -TraceRoute

Exemplo de Saída:


ComputerName           : google.com
RemoteAddress          : 142.250.190.78
InterfaceAlias         : Wi-Fi
SourceAddress          : 192.168.1.100
PingSucceeded          : True
TraceRoute             :
    Hop 1: 192.168.1.1 (5ms)
    Hop 2: 10.0.0.1 (10ms)
    Hop 3: 172.16.0.1 (20ms)
    Hop 4: 142.250.190.78 (25ms)

Explicação: Mostra o trajeto até google.com, útil para identificar gargalos na rede.

8. Nslookup

Conceito: O comando nslookup consulta servidores DNS para resolver nomes de domínio em endereços IP ou obter informações detalhadas, como registros MX (e-mail) ou NS (servidores de nomes). É essencial para diagnosticar problemas de resolução de nomes ou verificar configurações de DNS.

Exemplo:

nslookup google.com

Exemplo de Saída:


Server:  dns.google
Address:  8.8.8.8

Name:    google.com
Address:  142.250.190.78

Exemplo avançado:

nslookup -type=MX google.com

Exemplo de Saída:


Server:  dns.google
Address:  8.8.8.8

google.com    MX preference = 10, mail exchanger = smtp.google.com

Explicação: Consulta registros MX para verificar servidores de e-mail associados a google.com.

9. Get-NetIPAddress

Conceito: O cmdlet Get-NetIPAddress exibe os endereços IP (IPv4 e IPv6) associados às interfaces de rede, incluindo detalhes como prefixo, tipo de endereço (estático ou DHCP) e validade. É útil para verificar configurações de IP em interfaces específicas, como Wi-Fi, ou para automação em scripts.

Exemplo:

Get-NetIPAddress -InterfaceAlias "Wi-Fi"

Exemplo de Saída:


IPAddress         : 192.168.1.100
InterfaceAlias    : Wi-Fi
AddressFamily     : IPv4
Type              : Unicast
PrefixLength      : 24

Explicação: Mostra o endereço IP da interface Wi-Fi, útil para verificar configurações de rede.

10. Get-NetRoute

Conceito: O cmdlet Get-NetRoute exibe a tabela de roteamento do sistema, que define como os pacotes são encaminhados para diferentes redes. Inclui informações como destino, gateway, métrica e interface usada. É essencial para diagnosticar problemas de roteamento ou configurar redes complexas.

Exemplo:

Get-NetRoute

Exemplo de Saída:


DestinationPrefix    NextHop         InterfaceAlias    Metric
-----------------    -------         --------------    ------
0.0.0.0/0            192.168.1.1     Wi-Fi             25
192.168.1.0/24       0.0.0.0         Wi-Fi             0

Explicação: Lista rotas de rede, mostrando o gateway padrão usado pela interface Wi-Fi.

11. Get-NetFirewallRule

Conceito: O cmdlet Get-NetFirewallRule lista as regras do firewall do Windows, incluindo informações sobre portas, protocolos, direção do tráfego (entrada/saída) e status (habilitado/desabilitado). É útil para verificar permissões de rede ou diagnosticar bloqueios de tráfego.

Exemplo:

Get-NetFirewallRule -Enabled True

Exemplo de Saída:


Name                  : HTTP-In
DisplayName           : HTTP
Enabled               : True
Direction             : Inbound
Action                : Allow

Explicação: Lista apenas as regras de firewall ativas, útil para verificar permissões de tráfego.

12. Get-NetConnectionProfile

Conceito: O cmdlet Get-NetConnectionProfile exibe o perfil de conexão de rede (Público, Privado ou Domínio), que determina as configurações de firewall aplicadas. É importante para garantir que a interface (como Wi-Fi) esteja configurada com o perfil correto para segurança e conectividade.

Exemplo:

Get-NetConnectionProfile -InterfaceAlias "Wi-Fi"

Exemplo de Saída:


Name             : Network
InterfaceAlias   : Wi-Fi
NetworkCategory  : Private
IPv4Connectivity : Internet

Explicação: Mostra o perfil da conexão Wi-Fi, útil para ajustar configurações de firewall.

13. Get-NetAdapterAdvancedProperty

Conceito: O cmdlet Get-NetAdapterAdvancedProperty exibe propriedades avançadas de uma placa de rede, como configurações de velocidade, duplex, VLAN ou opções específicas do driver. É útil para otimizar o desempenho da rede ou diagnosticar configurações incorretas.

Exemplo:

Get-NetAdapterAdvancedProperty -Name "Wi-Fi"

Exemplo de Saída:


Name            DisplayName                DisplayValue
----            -----------                ------------
Wi-Fi           Speed & Duplex             Auto Negotiation
Wi-Fi           Preferred Band             5 GHz

Explicação: Lista propriedades avançadas da placa Wi-Fi, como a banda preferida (2.4 GHz ou 5 GHz).

14. Get-NetAdapterChecksumOffload

Conceito: O cmdlet Get-NetAdapterChecksumOffload verifica as configurações de descarregamento de checksum, que permitem à placa de rede processar verificações de integridade de pacotes, reduzindo a carga na CPU. É importante para redes de alta performance.

Exemplo:

Get-NetAdapterChecksumOffload -Name "Wi-Fi"

Exemplo de Saída:


Name            TcpIPv4Checksum    UdpIPv4Checksum    TcpIPv6Checksum
----            ---------------    ---------------    ---------------
Wi-Fi           Enabled            Enabled            Enabled

Explicação: Mostra se o descarregamento de checksum está ativo para a interface Wi-Fi.

15. Get-NetAdapterRss

Conceito: O cmdlet Get-NetAdapterRss exibe configurações de Receive Side Scaling (RSS), que distribui o processamento de pacotes de rede entre múltiplos núcleos da CPU. É útil para melhorar o desempenho em servidores ou dispositivos com alto tráfego de rede.

Exemplo:

Get-NetAdapterRss -Name "Wi-Fi"

Exemplo de Saída:


Name            Enabled    NumberOfReceiveQueues
----            -------    ---------------------
Wi-Fi           True       2

Explicação: Verifica se o RSS está habilitado na interface Wi-Fi, útil para otimização de desempenho.

16. Get-DnsClient

Conceito: O cmdlet Get-DnsClient exibe configurações do cliente DNS, como sufixos de pesquisa e comportamento de resolução de nomes. É útil para verificar como o sistema resolve nomes de domínio em redes locais ou corporativas.

Exemplo:

Get-DnsClient

Exemplo de Saída:


InterfaceAlias    ConnectionSpecificSuffix
--------------    ------------------------
Wi-Fi             home
Ethernet          company.local

Explicação: Lista sufixos DNS para todas as interfaces, úteis para configurar resolução de nomes.

17. Get-DnsClientServerAddress

Conceito: O cmdlet Get-DnsClientServerAddress mostra os endereços dos servidores DNS configurados para cada interface de rede. É essencial para verificar se o sistema está usando servidores DNS corretos, como os fornecidos pelo provedor ou públicos (ex.: 8.8.8.8).

Exemplo:

Get-DnsClientServerAddress -InterfaceAlias "Wi-Fi"

Exemplo de Saída:


InterfaceAlias    ServerAddresses
--------------    ---------------
Wi-Fi             {8.8.8.8, 8.8.4.4}

Explicação: Mostra os servidores DNS configurados para a interface Wi-Fi.

18. Resolve-DnsName

Conceito: O cmdlet Resolve-DnsName é a alternativa PowerShell ao nslookup, permitindo consultas DNS detalhadas, como resolução de nomes, registros MX, TXT ou CNAME. Ele retorna objetos estruturados, ideais para automação e análise.

Exemplo:

Resolve-DnsName google.com

Exemplo de Saída:


Name             Type   TTL   Section    IPAddress
----             ----   ---   -------    ---------
google.com       A      300   Answer     142.250.190.78

Explicação: Resolve o nome google.com em seu endereço IP.

19. Get-NetAdapterPowerManagement

Conceito: O cmdlet Get-NetAdapterPowerManagement exibe configurações de gerenciamento de energia da placa de rede, como opções de economia de energia ou wake-on-LAN. É útil para otimizar o consumo de energia ou garantir que a placa permaneça ativa em cenários críticos.

Exemplo:

Get-NetAdapterPowerManagement -Name "Wi-Fi"

Exemplo de Saída:


Name            WakeOnLan    PowerSaving
----            ---------    -----------
Wi-Fi           Enabled      Enabled

Explicação: Mostra configurações de energia da interface Wi-Fi, úteis para laptops.

20. Get-NetAdapterSriov

Conceito: O cmdlet Get-NetAdapterSriov verifica o suporte a SR-IOV (Single Root I/O Virtualization), uma tecnologia que permite que uma placa de rede seja compartilhada diretamente com máquinas virtuais, melhorando o desempenho em ambientes virtualizados.

Exemplo:

Get-NetAdapterSriov -Name "Wi-Fi"

Exemplo de Saída:


Name            SriovSupport
----            ------------
Wi-Fi           NotSupported

Explicação: Verifica se a interface Wi-Fi suporta SR-IOV (geralmente não suportado em adaptadores Wi-Fi).

21. Get-NetAdapterRdma

Conceito: O cmdlet Get-NetAdapterRdma verifica o suporte a RDMA (Remote Direct Memory Access), uma tecnologia que permite transferências de dados diretamente entre memórias de computadores, reduzindo a latência em redes de alta performance, como em data centers.

Exemplo:

Get-NetAdapterRdma -Name "Wi-Fi"

Exemplo de Saída:


Name            RdmaEnabled
----            -----------
Wi-Fi           False

Explicação: Mostra se RDMA está habilitado na interface Wi-Fi (geralmente não suportado).

22. Get-NetAdapterBinding

Conceito: O cmdlet Get-NetAdapterBinding exibe os protocolos de rede vinculados a uma placa, como IPv4, IPv6 ou protocolos de virtualização. É útil para verificar se uma interface está configurada para usar os protocolos corretos.

Exemplo:

Get-NetAdapterBinding -Name "Wi-Fi"

Exemplo de Saída:


Name            DisplayName            ComponentID    Enabled
----            -----------            -----------    -------
Wi-Fi           IPv4                   ms_tcpip       True
Wi-Fi           IPv6                   ms_tcpip6      True

Explicação: Lista os protocolos ativos na interface Wi-Fi, como IPv4 e IPv6.

23. Get-NetIPInterface

Conceito: O cmdlet Get-NetIPInterface exibe configurações de interface IP, como MTU (Maximum Transmission Unit), tipo de endereço (DHCP ou estático) e opções de encaminhamento. É útil para ajustar configurações de rede ou diagnosticar problemas de conectividade.

Exemplo:

Get-NetIPInterface -InterfaceAlias "Wi-Fi"

Exemplo de Saída:


InterfaceAlias    AddressFamily    Dhcp    Mtu
--------------    -------------    ----    ---
Wi-Fi             IPv4             Enabled 1500

Explicação: Mostra configurações IP da interface Wi-Fi, como uso de DHCP e tamanho do MTU.

24. Get-NetCompartment

Conceito: O cmdlet Get-NetCompartment lista os compartimentos de rede, que são usados para isolar configurações de rede em sistemas com múltiplos contextos (ex.: contêineres ou VPNs). Cada compartimento tem um ID único e configurações específicas.

Exemplo:

Get-NetCompartment

Exemplo de Saída:


CompartmentId    CompartmentDescription
-------------    ----------------------
1                Default Compartment

Explicação: Lista compartimentos de rede, úteis em sistemas complexos com virtualização.

25. Get-NetOffloadGlobalSetting

Conceito: O cmdlet Get-NetOffloadGlobalSetting exibe configurações globais de descarregamento de rede, como checksum, RSS e LSO (Large Send Offload). Essas configurações afetam todas as placas de rede e são usadas para otimizar o desempenho.

Exemplo:

Get-NetOffloadGlobalSetting

Exemplo de Saída:


ReceiveSideScaling    : Enabled
LargeSendOffload      : Enabled
ChecksumOffload       : Enabled

Explicação: Mostra configurações globais de descarregamento, úteis para otimização de desempenho.

26. Get-NetAdapterStatistics

Conceito: O cmdlet Get-NetAdapterStatistics exibe estatísticas de tráfego de uma placa de rede, como pacotes enviados, recebidos, erros e descartes. É útil para monitorar o desempenho da rede ou diagnosticar problemas de tráfego.

Exemplo:

Get-NetAdapterStatistics -Name "Wi-Fi"

Exemplo de Saída:


Name            ReceivedBytes    SentBytes    Errors
----            -------------    ---------    ------
Wi-Fi           123456789        98765432     0

Explicação: Mostra estatísticas de tráfego da interface Wi-Fi, útil para monitoramento.

27. Get-NetAdapterEncapsulatedPacketTaskOffload

Conceito: O cmdlet Get-NetAdapterEncapsulatedPacketTaskOffload verifica configurações de descarregamento de pacotes encapsulados, usados em túneis de rede como VXLAN ou GRE. Isso reduz a carga na CPU ao processar pacotes encapsulados.

Exemplo:

Get-NetAdapterEncapsulatedPacketTaskOffload -Name "Wi-Fi"

Exemplo de Saída:


Name            VxlanOffload    GreOffload
----            ------------    ----------
Wi-Fi           NotSupported    NotSupported

Explicação: Mostra se a interface Wi-Fi suporta descarregamento de pacotes encapsulados.

28. Get-NetAdapterHardwareInfo

Conceito: O cmdlet Get-NetAdapterHardwareInfo fornece detalhes de hardware da placa de rede, como slot PCI, versão do driver e informações do fabricante. É útil para verificar compatibilidade ou diagnosticar problemas de hardware.

Exemplo:

Get-NetAdapterHardwareInfo -Name "Wi-Fi"

Exemplo de Saída:


Name            SlotNumber    DriverVersion
----            ----------    -------------
Wi-Fi           0             20.70.0.5

Explicação: Mostra informações de hardware da interface Wi-Fi, como a versão do driver.

29. Get-NetAdapterQos

Conceito: O cmdlet Get-NetAdapterQos exibe configurações de Qualidade de Serviço (QoS), que controlam a priorização de tráfego de rede. É usado para garantir que tipos específicos de tráfego (ex.: VoIP) tenham prioridade em redes congestionadas.

Exemplo:

Get-NetAdapterQos -Name "Wi-Fi"

Exemplo de Saída:


Name            Enabled    DefaultPriority
----            -------    ---------------
Wi-Fi           True       3

Explicação: Mostra configurações QoS da interface Wi-Fi, úteis para otimização de tráfego.

30. Get-NetAdapterVPort

Conceito: O cmdlet Get-NetAdapterVPort exibe configurações de portas virtuais, usadas em ambientes de virtualização para conectar máquinas virtuais a redes físicas. É relevante em cenários de servidores com Hyper-V ou outras tecnologias de virtualização.

Exemplo:

Get-NetAdapterVPort -Name "Wi-Fi"

Exemplo de Saída:


Name            VPortId    Enabled
----            -------    -------
Wi-Fi           None       False

Explicação: Mostra se portas virtuais estão configuradas na interface Wi-Fi (geralmente não aplicável).
